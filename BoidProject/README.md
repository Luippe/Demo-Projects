## Controls

| Input | Action |
|-------|--------|
| Left-click | Add a boid |
| Right-click | Add an obstacle |
| `W` / `A` / `S` / `D` | Move the view |
| `Space` | Toggle the visible screen borders (on by default) |
| Close the window | Quit |

A [boids](https://en.wikipedia.org/wiki/Boids) flocking simulation written in
**Java** with Swing. Agents follow the classic separation / alignment / cohesion
rules to produce emergent flocking behaviour, with obstacles and a Voronoi
component in the mix.

> This is the only **Java** project in the collection — it does not use Python
> and has no `requirements.txt`.

## Requirements

- **JDK 17 or newer** (`javac` and `java` on your PATH). Check with:
  ```bash
  java -version
  ```

No external libraries are needed (`lib/` is empty) — it's plain Swing.

## Run it

From inside the `BoidProject` folder:

```powershell
# 1. Compile the sources into bin/
$sources = Get-ChildItem src\main\*.java, src\entity\*.java
javac -d bin $sources.FullName

# 2. Run the Main class
java -cp bin main.Main
```

The window opens maximized. **Close the window to exit.**

## Build the Windows application

Run PowerShell inside the `BoidProject` folder:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\Build-Boids.ps1
```

This creates:

- `..\dist\Boids\Boids.exe` — the portable launcher. Keep the whole `Boids`
  folder together because it contains the bundled Java runtime.
- `..\dist\Boids-Windows.zip` — the portable folder ready to share.

To create a single downloadable Windows installer, install WiX and run:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\Build-Boids.ps1 -Type Installer
```

The installer is written to `..\dist\Boids.exe`, matching the GitHub Release
download button above. The installer and portable application are self-contained.
Every build also compiles with Java warnings enabled and runs an automated
simulation/rendering smoke test before packaging. The script automatically finds
WiX in its standard installation folders even when WiX is not on `PATH`.

`..\dist\Boids.exe` is the installer, not the simulation launcher. Run it once,
then start the simulation using the **Boids** desktop shortcut, the Start menu
entry, or `%LOCALAPPDATA%\Boids\Boids.exe`. Running the installer again does not
launch the simulation.

### Or use VS Code

Open this folder in VS Code with the *Extension Pack for Java* installed and press
**Run** — the included `.vscode/settings.json` already points source at `src/` and
output at `bin/`.

## Project layout

- `src/main/` — entry point (`Main.java`), screen/rendering, input handling, obstacles, Voronoi.
- `src/entity/` — `Entity`, `Player`, `Enemy` agent classes.
- `backup/` — saved settings and entity snapshots (plain text).
- `test/` — dependency-free simulation and rendering smoke test.
- `bin/` — compiled `.class` output (git-ignored; regenerated by `javac`).
